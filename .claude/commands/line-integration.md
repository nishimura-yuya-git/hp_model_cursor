# 📱 LINE統合開発マニュアル（安全・テスト計画）

## 📋 開発着手前の必須確認事項

### 1. 必要な環境変数・設定
LINE Messaging APIの開発には、以下の情報が必要です。
- **`LINE_CHANNEL_ID`**: チャンネルの基本設定から取得
- **`LINE_CHANNEL_SECRET`**: チャンネルの基本設定から取得
- **`LINE_CHANNEL_ACCESS_TOKEN`**: メッセージ送信に必須（Messaging API設定から発行）
- **`Webhook URL`**: LINEからのイベント（返信、ボタン操作等）を受信するために設定必須。システム側のエンドポイント（例: `https://.../api/line/webhook`）を指定します。

### 2. 安全なテスト計画（西村様限定）
開発初期段階およびテストフェーズにおいては、**誤送信や無限ループによる全ユーザーへの通知を防ぐため**、以下の制限を徹底します。

- **最優先テスト対象**: `id: 47` (西村様)
- **原則**: 開発・検証が完了するまで、全ユーザーへの一斉送信（ブロードキャスト）機能の実行は厳禁。
- **ガード実装**: 送信ロジックには必ず `if (userId === '47')` 等のチェックを入れ、意図しないユーザーへの送信を防止します。

### 3. 無限ループ・過剰API実行の防止
- **Webhookループの遮断**: 自身の返信に反応して再度Webhookをトリガーする無限ループを防止するため、ボット自身のIDを判定して無視するロジックを実装します。
- **レート制限への配慮**: 短時間での過剰なAPI呼び出しを避け、エラー発生時は指数バックオフ（Exponential Backoff）を用いたリトライ制御を行います。

`.cursor/commands` 内の既存ドキュメント（`security.md`, `resend-email.md`, `supabase-cms.md` 等）を調査し、以下の安全性を確認しました。

- **ユーザー処理の有無**: 現在のドキュメントには、西村様（ID: 47）以外の特定のユーザーを自動処理するような記述やロジックは含まれていません。
- **API実行・無限ループ**: 現状のファイル群は規約やテンプレートであり、無限ループやAPIの過剰実行を引き起こすようなコード片は存在しません。
- **APIテスト**: 各機能はモジュール化されており、個別にAPIテスト（西村様のアカウント宛）が可能な設計を目指します。
