# 💀 Skeleton Loading Skill

## 概要
データ取得中やページリロード時に、コンテンツの構造を模した「スケルトン（プレースホルダー）」を表示し、ユーザーに読み込み中であることを視覚的に伝えるスキルです。

## 実装ガイドライン

### 1. スケルトンコンポーネントの使用
- `src/components/ui/Skeleton/Skeleton.tsx` に定義されている共通の `Skeleton` コンポーネントを使用してください。
- 表示される実コンテンツの形状（長方形、円、テキスト行）に合わせて `variant` を使い分けてください。
  - `rect`: カードや画像、大きなブロック要素
  - `circle`: アイコンやアバター
  - `text`: テキストの行（デフォルトの高さが設定されています）

### 2. レイアウトの維持
- スケルトンは最終的なコンテンツとほぼ同じレイアウト（高さ、幅、余白）で配置してください。読み込み完了時のガタつき（レイアウトシフト）を最小限に抑えます。

### 3. ローディング状態の制御
- コンポーネント内で `loading` ステートを管理し、データ取得中はスケルトンを、取得完了後は実データを表示するように条件分岐を行ってください。

```tsx
// 実装例
{loading ? (
  <div className="space-y-4">
    <Skeleton variant="text" width="60%" />
    <Skeleton variant="rect" height={200} />
  </div>
) : (
  <div className="space-y-4">
    <h2 className="text-xl font-bold">{data.title}</h2>
    <img src={data.imageUrl} className="w-full h-[200px] object-cover rounded-xl" />
  </div>
)}
```

### 4. 複数要素のスケルトン
- リスト形式のデータを読み込む場合は、`Array.from({ length: N }).map(...)` を使用して、複数行のスケルトンを表示してください。

## 完了定義（DoD）
- 読み込み中に白い画面や唐突なスピナーだけが表示されるのではなく、コンテンツの枠組みが見えていること。
- アニメーション（`animate-pulse`）が適用され、システムが動作していることが伝わること。
- デザインシステム（余白、角丸）に準拠したスケルトンであること。
