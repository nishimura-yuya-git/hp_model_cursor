# 📋 要件定義書管理フォルダ

このフォルダでは、HP作成プロジェクトの要件定義書を業種別・プロジェクト別に管理します。

## 📁 フォルダ構成

```
requirements/
├── README.md                 # このファイル（使い方説明）
├── templates/               # 要件定義書テンプレート
│   ├── cafe-template.md     # カフェHP用テンプレート
│   ├── gym-template.md      # ジムHP用テンプレート
│   ├── beauty-template.md   # 美容院HP用テンプレート
│   ├── dental-template.md   # 歯科医院HP用テンプレート
│   └── general-template.md  # 一般企業HP用テンプレート
├── projects/               # 実際のプロジェクト要件定義書
│   ├── cafe-lumiere/       # カフェ・ルミエール（例）
│   ├── fitness-prime/      # フィットネス・プライム（例）
│   └── salon-elegance/     # サロン・エレガンス（例）
└── drafts/                 # 作成中の要件定義書
```

## 🚀 使い方

### 1. 新しいプロジェクト開始時

1. `templates/` から適切な業種テンプレートをコピー
2. `projects/プロジェクト名/` フォルダを作成
3. テンプレートをベースに要件定義書を作成
4. AI開発時に該当フォルダを参照

### 2. 要件定義書作成手順

```bash
# 例：カフェプロジェクトの場合
1. templates/cafe-template.md をコピー
2. projects/cafe-project-name/ フォルダ作成
3. requirements.md として保存
4. 追加資料があれば同フォルダに配置
```

### 3. AI開発での活用

```
「requirements/projects/プロジェクト名/requirements.md を参照してHPを作成してください」
```

このように指示することで、要件定義に基づいた正確なHP開発が可能になります。

## 📝 要件定義書の必須項目

- **プロジェクト概要**：目的・目標・ターゲット
- **デザイン要件**：カラー・フォント・スタイル
- **機能要件**：必要な機能・セクション
- **コンテンツ要件**：テキスト・画像・動画
- **技術要件**：使用技術・レスポンシブ対応
- **SEO要件**：キーワード・メタデータ
- **納期・予算**：スケジュール・制約事項

## 🎯 ベストプラクティス

1. **具体的な記述**：曖昧な表現を避け、具体的に記載
2. **参考サイト**：デザインの参考サイトURL記載
3. **画像・素材**：使用予定の画像や素材を明記
4. **優先順位**：機能の優先順位を明確化
5. **更新履歴**：変更があった場合の履歴管理
