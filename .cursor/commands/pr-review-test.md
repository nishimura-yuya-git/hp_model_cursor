# PR レビュー対応＋テスト検証コマンド

## 実行前に読むルール

```
.cursor/rules/pr-review-test.mdc
```

---

## プロンプトテンプレート（コピペ用）

### パターン1: PR番号を指定（GitHub MCP あり）

```
PR #{番号} のレビューコメントを GitHub MCP で取得して。
各指摘に対して：
1. 修正方針を説明
2. 実装を修正
3. Playwright（cursor-ide-browser）で E2E テストを実行して全件グリーンにして
4. 結果サマリーを報告して
```

**例**: `PR #42 のレビューコメントを...` のように番号を入れる

---

### パターン2: レビューコメントを貼り付け（GitHub MCP なし）

```
以下のPRレビュー指摘に対応して。修正後、PlaywrightでE2Eテストを実行して検証して。

【指摘1】
（ここにコメント内容を貼り付け）

【指摘2】
（ここにコメント内容を貼り付け）
```

---

### パターン3: 簡易版（実装＋テストのみ）

```
PR #{番号} のレビュー指摘を修正して、該当画面の動作確認をPlaywrightで実行して。
```

---

## 実行チェックリスト

```
□ pr-review-test.mdc のルールを確認したか
□ GitHub MCP が有効か（PR取得・コメント返信が必要な場合）
□ 開発サーバー（npm run dev）が起動しているか
□ テストデータのクリーンアップを実施するか
```

---

## リポジトリ情報（必要に応じて指定）

- **owner/repo**: プロジェクトのGitHubリポジトリ
- **PR番号**: 対象のプルリクエスト番号

GitHub MCP で PR を取得する際、現在開いているプロジェクトのリポジトリが自動的に使用される場合がある。別リポジトリの場合は明示的に指定すること。
