# Playwright MCP テストコマンド（汎用）

## 実行前に必ず読む

```
.cursor/skills/playwright-mcp-testing/SKILL.md
```

汎用的なテスト手順・クリーンアップルール・ツール一覧はスキルを参照する。

---

## 1. 事前確認

テスト開始前に、以下を確認する。

```text
□ テスト対象のURLまたは画面名は明確か
□ 開発サーバーが必要な場合、起動コマンドとポートは分かっているか
□ ログインが必要か
□ テストでデータを作成・編集・削除するか
□ テスト後のクリーンアップ方法が分かっているか
```

## 2. ログインが必要な場合の確認

ログインが必要な画面をテストする場合、**操作を始める前に必ずユーザーへ確認する**。

```text
この画面はログインが必要です。以下を教えてください。

- ログインURL
- メールアドレスまたはユーザーID
- パスワード
- 利用する権限・ロール（管理者 / スタッフ / 顧客など）
- テストで作成してよいデータ範囲
- テスト後に削除・復元すべきデータ
```

### 注意

- パスワードの変更・リセットは絶対に行わない。
- 認証情報をファイルへ保存しない。
- `.env` やSecretsを勝手に編集しない。
- 既にブラウザでログイン済みの場合も、対象ロールが正しいか確認する。

---

## 3. MCPツールの基本手順

1. `browser_tabs` で開いているタブを確認する。
2. 必要に応じて `browser_navigate` で対象URLへ移動する。
3. `browser_snapshot` で画面構造を確認する。
4. ref を使ってクリック・入力・選択を行う。
5. 操作後は再度 `browser_snapshot` で状態を確認する。
6. コンソールエラーやネットワーク失敗が疑われる場合は、該当ツールで確認する。

スクリーンショット確認が必要な場合のみ、`browser_take_screenshot` を使う。

---

## 4. テストデータの原則

- テストデータには `[TEST]` など、後から識別できる接頭辞を付ける。
- 本番データ・実ユーザーデータを破壊しない。
- 作成したデータは、テスト完了後に削除または元の状態へ戻す。
- 削除順序や関連テーブルが不明な場合は、先に設計・DB構造を確認する。

---

## 5. 報告フォーマット

テスト完了後は以下を簡潔に報告する。

```markdown
## 動作確認結果

- 対象画面:
- 実施した操作:
- 確認できたこと:
- 発見した問題:
- クリーンアップ:
- 残リスク:
```

問題が見つかった場合は、再現手順・期待値・実際の結果を分けて記録する。
