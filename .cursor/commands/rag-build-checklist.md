# RAG 構築チェックリスト（厳格版）適用コマンド

## ゴール

- **`doc/todo/rag-build-checklist.md`** を**唯一の正**として、現状コード・PR差分・運用が **セクション 0（不変条件）** と **該当フェーズ（P0/P1/P2）** を満たすか検証する。
- **［必須］** 未充足は**ブロッカー**として列挙し、例外があるなら**承認と Issue 番号**を要求する。

## 実行手順（エージェントが必ず辿る）

### Step 1: 一次資料の読込

1. `doc/todo/rag-build-checklist.md` を**全文**読む（要約に置き換えない）。
2. ユーザー指示または文脈から対象フェーズを決める：**P0（現状）** / **P1（RAG最小）** / **P2（本番品質）**。不明なら **P0 を最低ライン**として検証。

### Step 2: コード突合（実装の正）

次のファイルを開き、チェックリスト **セクション 7** と **セクション 0（I-1〜I-3）** と突合する。

- `server/manual-chat-route.js` — `streamText`、環境変数、`pipeTextStreamToResponse`
- `dev-api-server.js` — ルート登録
- `src/services/manualChat/streamManualChat.ts` — クライアント fetch・エラー分岐
- `src/pages/Manual/hooks/useManualChat.ts` — 送信ガード、フォールバック、無限再入の有無

### Step 3: セクション 5（API・安全性）の静的確認

- `useEffect` による API 連鎖、`onDelta` からの再送信、サーバー側の再帰呼び出しの**有無**を明示する。
- ツール／`maxSteps` を**新規導入**している差分がある場合、**上限設定**の有無を必ず確認する。

### Step 4: 出力フォーマット（厳守）
