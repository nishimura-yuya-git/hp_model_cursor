# Playwright MCP テストスキル（汎用）

## 概要

`user-playwright` MCP サーバーを使い、実際のブラウザ上でアプリの動作を確認する。
テストで作成・編集したデータは **必ずテスト後に元の状態に戻す**。

プロジェクト固有の情報（URL・ページ一覧・認証情報）は
`.cursor/commands/playwright-mcp-test.md` を参照すること。

---

## 基本ワークフロー

```
1. 開発サーバーの起動確認
2. ブラウザ起動 → ログイン状態の確認
3. テスト前の状態をスナップショットで記録
4. テスト対象ページへ移動・操作
5. スナップショット・スクリーンショットで結果確認
6. テストデータのクリーンアップ（登録・編集した場合）
7. 結果レポート
```

---

## Step 1: 開発サーバー確認

```
terminals/ フォルダの .txt ファイルを head -n 10 で確認
→ last_command に "dev" や "vite" があれば起動中
→ 起動していなければ: npm run dev をバックグラウンド実行（block_until_ms: 0）
```

---

## Step 2: MCPツールの使い方

### ツール一覧

| ツール | 用途 |
|--------|------|
| `browser_navigate` | URLに移動 |
| `browser_snapshot` | ページ構造取得（操作前に必ず実行） |
| `browser_click` | ref番号でクリック |
| `browser_fill` | テキスト入力（クリア後に入力） |
| `browser_type` | テキスト追記入力 |
| `browser_select_option` | セレクトボックス選択 |
| `browser_take_screenshot` | 視覚確認用スクリーンショット |
| `browser_wait_for` | 要素の出現・消滅待ち |
| `browser_press_key` | キーボード操作（Enter/Escape/End等） |
| `browser_tabs` | タブ一覧確認 |

### 操作の基本原則

1. **操作前に必ず `browser_snapshot`** を実行して最新の ref を取得する
2. ref は動的に変わるため、古い ref でクリックしない
3. モーダルを開いた後も再度 `browser_snapshot` を実行する
4. 確認ポイントでは `browser_take_screenshot` を撮影する

---

## Step 3: テストデータ管理（最重要）

### テスト開始前の準備

```
1. 対象ページのスナップショット・スクリーンショットを取得
2. 既存データの件数・内容・IDをメモ
3. テストで追加するデータには「[TEST]」プレフィックスを付ける
   例: 名前 → "[TEST] テスト用データ"
```

### テスト後のクリーンアップ（必須）

```
1. [TEST] プレフィックスのデータを特定
2. 削除操作を実行
3. 編集したデータは事前記録の値に戻す
4. クリーンアップ後にスナップショットで「テスト前と同じ状態」を確認
```

### クリーンアップ対応表

| 操作 | クリーンアップ方法 |
|------|-----------------|
| 新規データ追加 | 追加したデータを削除 |
| 既存データ編集 | 元の値に戻す（事前記録必須） |
| 設定・フラグ変更 | 元の値に戻す |
| ファイルアップロード | アップロードしたファイルを削除 |
| 読み取り・閲覧のみ | 不要 |

---

## Step 4: よくある操作パターン

### モーダルを開いて確認する

```
1. browser_snapshot でページ構造確認・ボタンのref取得
2. browser_click でモーダルを開く
3. browser_snapshot でモーダル内容確認
4. browser_take_screenshot で視覚確認
5. 変更しない場合はキャンセル/Escape で閉じる
```

### フォームに入力して保存する

```
1. モーダルを開く
2. browser_snapshot で入力フィールドのref確認
3. browser_fill または browser_type で入力
4. 保存ボタンをクリック
5. 成功/エラーメッセージを確認
6. ★ クリーンアップを実施
```

### セレクト・ラジオ・チェックボックス

```
1. browser_snapshot で選択肢（options）を確認
2. browser_select_option で選択（セレクト）
3. browser_click でクリック（ラジオ・チェック）
4. 変更後のスナップショットで反映確認
```

### スクロールが必要な場合

```
browser_press_key で "End" または "PageDown"
または browser_scroll でスクロール
```

## Step 5: テスト結果レポート形式

```
## テスト結果

### ✅ 正常動作確認
- [機能名]: [確認内容] → 正常

### ⚠️ 注意事項
- [気になった点]

### ❌ 問題あり（あれば）
- [問題の説明]
- スクリーンショット: [ファイルパス]

### 🧹 クリーンアップ
- [TEST]データ削除: ✅ 完了
- 編集データ復元: ✅ 完了
```

## エラー対処

| エラー | 対処 |
|--------|------|
| ページが404 | URLを確認、ルーティング設定を確認 |
| ref が見つからない | `browser_snapshot` を再実行して最新のrefを取得 |
| ログインが切れている | ログインページへ移動して再認証 |
| タイムアウト | `browser_wait_for` で要素の出現を待つ |
| 出力が大きすぎる | ファイルに書き出されるので Read ツールで読む |
| Edge Function接続エラー | ローカル環境では想定内（無視可） |
